部署网络处理手段

从流量形状出发，系统学习反向代理、正向代理、SSH Tunnel、内网穿透、VPN/Overlay 和排障路径。

学习顺序

建议按编号学习。每节课都很短，重点是建立一个可复用的判断动作。

课程

0001 · 先判断流量形状，再选网络工具

建立总地图：谁访问谁、谁能主动连接、该选哪类工具。

总览选型

0002 · 反向代理、TLS 终止与真实客户端信息

理解公网入口、TLS 终止、后端监听地址和 X-Forwarded-*。

反向代理TLS

0003 · 正向代理、HTTP CONNECT 与 SOCKS5

理解出口代理、CONNECT 隧道、SOCKS5 和 DNS 在哪里解析。

正向代理SOCKS5

0004 · SSH Tunnel：-L、-R、-D 与跳板机

用“监听端口在哪里”区分本地转发、远程转发和动态转发。

SSHTunnel

0005 · 内网穿透：没有公网 IP 时如何暴露服务

理解 Origin → Relay ← User，以及 tunnel 不是自动安全。

内网穿透Relay

0006 · VPN / Overlay / Subnet Router

区分暴露单个服务和建立受控私网；理解 peer、路由、ACL、subnet router。

VPNOverlay

0007 · 部署网络排障：DNS → TCP → TLS → HTTP → 代理 → 应用

按层定位问题，不盲改配置；用 dig、curl -v、ss、tcpdump。

下一批建议

Docker / Kubernetes 网络

端口发布、bridge、Service、Ingress、LoadBalancer、Gateway。

CDN / Cloudflare / 边缘网络

DNS 托管、缓存、源站保护、边缘 TLS、回源。

Identity-aware proxy、设备姿态、ACL、mTLS、应用身份边界。

真实案例排障

502、证书错、Docker 端口不可达、Cloudflare 522/525、内网 DNS。

学习建议：每节课看完后，用自己的一个真实服务场景画出路径图：客户端、DNS、代理、隧道、服务、TLS 终止点、认证点。
