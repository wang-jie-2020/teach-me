Lesson 0003 · Forward Proxy

正向代理、HTTP CONNECT 与 SOCKS5

本课练一个判断：代理到底是在帮“客户端出网”，还是在帮“服务端接客”。正向代理解决的是出口控制，不是入口暴露。

这一课的胜利标准

你能区分 HTTP 正向代理、HTTP CONNECT 隧道、SOCKS5；并能用 curl 验证请求是否真的走了代理。

正向代理的形状

Client / App │ request to proxy ▼ Forward Proxy / Egress Gateway │ request to target ▼ Internet target

正向代理代表客户端访问目标站点。企业常用它做统一出口、访问控制、审计、缓存、下载加速、合规拦截。它回答的问题是：“谁可以从内网访问外部哪里？”

三种常见出口形态

HTTP 正向代理

HTTP可看见请求

curl → proxy → http://target

客户端把完整目标 URL 发给代理，代理代为请求。适合 HTTP 流量。

HTTP CONNECT

HTTPS隧道

client → CONNECT host:443 → proxy ⇄ target

代理先建立到目标的 TCP 隧道，之后盲转发字节流，常用于通过代理访问 HTTPS。

SOCKS5

通用 TCP应用指定目标

app → socks5://proxy → target

比 HTTP 代理更通用。SSH -D 会在本机开一个 SOCKS4/5 代理。

DNS 在哪里解析？

这点很容易踩坑：使用 SOCKS 时，域名可以在本机解析，也可以交给代理端解析。

# socks5：通常本机先解析目标域名，再把 IP 给代理
curl -v --proxy socks5://127.0.0.1:1080 https://example.com/

# socks5h：把 hostname 交给 SOCKS 代理端解析
curl -v --proxy socks5h://127.0.0.1:1080 https://example.com/

如果目标域名只在远端内网 DNS 可解析，常需要 socks5h。

验证请求走代理

# HTTP/HTTPS 经 HTTP 代理
curl -v -x http://proxy.example.com:3128 https://example.com/

# 本机 SSH 动态转发开 SOCKS5
ssh -D 127.0.0.1:1080 user@jump-host
curl -v --proxy socks5h://127.0.0.1:1080 https://example.com/

# 查看 curl 输出中的关键线索：
# * Uses proxy env variable ...
# * Trying 127.0.0.1:1080...
# * CONNECT tunnel established

安全边界

  • 不要开放一个“任意人可用”的出口代理，它会变成匿名跳板。
  • 支持 CONNECT 的代理应限制目标端口和目标范围。RFC 9110 明确提醒，任意 CONNECT 到保留端口可能被滥用。
  • 出口代理能记录元数据，甚至在 TLS 检查场景中解密流量；要有明确的授权和告知。

练习

场景 1

公司要求所有后端服务访问公网 API 都必须经过统一出口，并记录访问日志。选哪类方案？

场景 2

你通过 SSH -D 访问远端内网网站 admin.internal，本机 DNS 解析不到。curl 该优先用什么？

推荐主源

读 everything curl: Proxies 学会如何从客户端验证代理；读 RFC 9110 CONNECT 理解 HTTP CONNECT 的隧道边界。
