Lesson 0006 · Overlay Network

VPN / Overlay / Subnet Router

本课练一个判断：什么时候该暴露“一个服务”，什么时候该建立“一张受控私网”。

这一课的胜利标准

你能解释 WireGuard/Tailscale 这类 Overlay 解决的不是 HTTP 入口，而是“节点之间如何带身份、带路由地互访”。

Overlay 的形状

Laptop 100.x.y.1 │ ├── encrypted overlay network + ACL │ Server 100.x.y.2 ── Subnet Router ── 10.0.0.0/24

反向代理和内网穿透通常暴露某个服务；Overlay/VPN 更像给设备建立一张虚拟网。加入这张网的节点获得虚拟 IP，再通过路由和 ACL 互访。

三个概念

Peer

加入虚拟网络的设备或节点。WireGuard 里 peer 由公钥识别；Tailscale 在此之上加了身份和控制面。

Allowed IPs / 路由

决定哪些目标网段走隧道。它既是路由选择，也常是访问范围声明。

Subnet Router

一个已加入 Overlay 的设备替传统子网发布路由，让没安装客户端的内网设备也能被访问。

什么时候选 Overlay

需求反代/穿透Overlay/VPN
公网用户访问一个网站合适通常不合适
员工访问多个内部系统会变成很多入口合适
跨云服务器互访数据库、缓存、管理口不够自然合适
临时给客户演示一个页面合适过重
访问整个办公室网段不合适Subnet Router 合适

Subnet Router 的直觉

假设办公室有一段传统内网 192.168.10.0/24，打印机、NAS、旧系统都不能装 Tailscale。你可以让一台 Linux 小主机加入 tailnet，并“广告”这段路由：

Your Laptop ── Tailnet ── Linux Router ── 192.168.10.0/24

Tailscale 官方文档把 subnet router 描述为 tailnet 中的网关，用来向其他设备发布传统子网路由。默认可能做 SNAT；如果要保留源 IP，需要理解并配置相关选项。

安全边界

  • Overlay 不是“全员全网互通”。必须用 ACL 限制谁能访问哪些主机/端口。
  • Subnet Router 会把一个传统网段带进虚拟网络，范围要尽量小。
  • 注意 SNAT：保留源 IP 有利于审计，但需要回程路由正确；SNAT 简单但会让后端只看到路由器 IP。
  • 设备信任很重要：丢失的笔记本可能仍拥有内网路径，需能快速撤销。
