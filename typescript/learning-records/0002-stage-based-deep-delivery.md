# 教学方式调整:整阶段深度产出

第二次会话,用户对首课反馈“内容偏少、太简单”,并要求改变交付方式。据此调整,记录以指导后续所有阶段。

- **深度要求**:课程要“深”——每节含多例子、常见坑(.warn)、3~5 个交互练习、代码挑战。首课(0001)已按此标准加深重做,作为深度模板锚点。
- **交付粒度**:按“阶段”整包产出,而非一次一课。一个阶段 = 完整掌握某个方向。用户已选定推进阶段 1→2→3。
- **阶段一蓝图(已定)**:0001 本质+注解+推断 / 0002 字面量+union+别名 / 0003 收窄 / 0004 对象 / 0005 数组元组枚举 / 0006 函数 / 0007 特殊类型+断言+综合复习。
- **含义**:后续阶段二(结构与泛型)、阶段三(高级类型)应沿用同一深度模板与整包交付。用户吸收快、要求高,ZPD 偏高,可放心加大单课信息密度,但仍保持每课可独立完成 + 即时反馈。
- 教学法上已加入:retrieval practice(输入判分)、interleaving(0007 综合交叉练习);后续阶段应继续用 spacing/interleaving 复习旧点。
