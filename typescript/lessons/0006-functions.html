函数类型深入

本课小胜利:能用"函数类型表达式 (a: T) => U"描述一段行为,会用 type 给函数类型起名,理解可选参数 / 默认参数 / 剩余参数的差异,看懂 void 返回值的特殊含义,会用调用签名和函数重载表达"可调用 + 多种调用方式"。约 24 分钟。

1 · 函数类型表达式:把"行为"也写成类型

在 0001 你已经会写"参数 + 返回值"的注解了。TypeScript 还允许把"一个函数本身"作为类型,语法是箭头函数形状(叫 function type expression):

// 形如: (参数列表) => 返回值
type Greet = (name: string) => string;

const sayHi: Greet = (n) => `Hi, ${n}`;
const sayHello: Greet = (n) => `Hello, ${n}`;

它和函数声明的写法是镜像关系,只是少了函数名、参数名也可以省(类型由类型表达式锁定):

// 写法 A:函数声明
function add(a: number, b: number): number { return a + b; }

// 写法 B:函数类型表达式 + 赋值
type Add = (a: number, b: number) => number;
const add: Add = (a, b) => a + b;

这种"行为即类型"的能力是函数式编程风格的支柱——它让你能把"如何执行"延后到赋值时决定,变量只承诺"我接什么、吐什么"。

来源:Handbook · More on Functions · Function Type Expressions。

常见坑 ①:(string) => void 不是"接 string 返回 void"

初学者最常踩的坑之一——这个写法看起来像"接 string 返回 void",但其实完全不是。看下面:

// 看起来像: (string) => void
function log(msg) { console.log(msg); }

// 真正的含义是:
//   参数名 叫 "string",类型是隐式 any
//   返回值类型也是隐式 any
// ↓ 正确写法 ↓
function log(msg: string): void { console.log(msg); }

// 作为类型表达式 ↓
type Log = (msg: string) => void;

记忆口诀:参数一定要写 name: type 两个东西,少一个就翻车。

2 · 给函数类型起名:type 复用 + 跨文件共享

把函数类型提取成别名,好处有三:

  1. 复用:同一份签名在多处使用,改一处全跟着改;
  2. 可读性:Parser 比 (input: string) => AST 更易沟通;
  3. 可导出:export type 让别的文件 import 进来用。
// —— types.ts ——
export type CompareFn = (a: number, b: number) => number;

// —— sort.ts ——
import type { CompareFn } from "./types";

function sort(arr: number[], cmp: CompareFn): number[] {
  return [...arr].sort(cmp);
}

注意:用 import type 引入的别名编译后会被擦除(0001 的 type erasure 同样适用),不会生成额外的运行时代码。

来源:Handbook · More on Functions。

3 · 参数的三大形态:必选 / 可选 / 默认 / rest

3.1 可选参数 ?

在参数名后加 ? 表示调用时可以省略。省略后该参数的值是 undefined——所以函数体内要把它当 T | undefined 处理。

function greet(name: string, prefix?: string): string {
  return prefix ? `${prefix} ${name}` : name;
}
greet("Ada");              // ✅ prefix 是 undefined
greet("Ada", "Dr.");       // ✅ 传了就用
greet("Ada", undefined); // ✅ 等同于省略

规则:可选参数必须在必选参数之后(否则位置对不上)。

3.2 默认参数

用 = 默认值 既能省事,又锁定了缺省行为:

function greet(name: string, prefix: string = "Hi"): string {
  return `${prefix} ${name}`;
}
greet("Ada");        // "Hi Ada"
greet("Ada", "Yo");  // "Yo Ada"

默认参数的位置不受限制(JS 早就有这个特性),但出于可读性通常还是放后面。

常见坑 ②:可选参数 ≠ 一定带默认值

看起来"给个默认值更香",但两者在类型层含义不同:

function a(x?: number) { return x; }      // 内部 x: number | undefined
function b(x: number = 0) { return x; }      // 内部 x: number(默认已注入)

不能同时写 x?: number = 0(语法错误,tsc 直接报错)。

经验:

  • 想让"不传就走默认值,函数体内不用判空" → 用 默认参数;
  • 想让"不传就显式 undefined,调用方必须面对这个事实" → 用 可选参数。

3.3 剩余参数 ...args: T[]

当参数数量不固定时,把"剩下的"放进数组。注意类型是数组 T[],调用时你可以传任意数量(包括 0 个):

function join(sep: string, ...parts: string[]): string {
  return parts.join(sep);
}
join(", ");                  // ""
join(", ", "a");            // "a"
join(", ", "a", "b", "c");  // "a, b, c"

规则:剩余参数必须是参数列表的最后一项,且只能有一个。在函数类型表达式里也支持:

type Logger = (prefix: string, ...args: unknown[]) => void;

来源:Handbook · More on Functions · Rest Parameters。

4 · 返回类型 void 的"特殊含义"

初学者的直觉:void = "这个函数啥也不返回"。更精确的说法是:函数的返回值会被忽略。 这带来一个非常特殊、也非常容易困惑的赋值兼容性——

// 1) 一个真正"不返回"的函数
function noop(): void { /* 没写 return */ }

// 2) 一个"返回 string"但调用方会忽略返回"的函数
type VoidFn = () => void;
const f: VoidFn = () => "I returned something";  // ✅ 不报错!

这是 TS 有意为之的设计:当一个函数类型被声明为 () => void 时,任何"返回任何东西"的实现都能赋给它。 这解决了回调地狱里一个真实痛点:

// 数组 forEach 的签名是 (cb: (item: T) => void) => void
// 如果 void 严格要求"不能返回",下面这行就会报错:
["a", "b"].forEach(s => s.toUpperCase()); // 箭头函数返回了 string

// 实际行为:不报错,返回值被静默忽略,代码可读性反而更好。

关键结论:

  • 实现侧:() => void 的实现可以返回任何值,调用方拿不到那个值;
  • 消费侧:反过来不成立——() => string 不能被 () => void 替换赋值,因为消费者依赖那个 string。

这条规则在 Handbook · Return type of void 里有官方解释。

返回 union 类型:把"成功/失败"都写在签名里

函数可以返回联合类型——这也是为什么 0005 的元组 [number, string] 在函数返回里特别常见:

// "可能是数字,也可能是 null"
function findAge(name: string): number | null {
  return name === "Ada" ? 36 : null;
}

// "成功返回数据,失败返回错误信息"——元组版
type Result = [ok: boolean, data: string];
function tryParse(s: string): Result {
  const n = Number(s);
  return [Number.isFinite(n), String(n)];
}

调用方在使用 union 返回值时,TS 会强制你"先收窄(0002 讲的类型守卫)再使用"——这就是把"可能出错"写进类型的能力。

5 · 调用签名 call signature:对象又能调用

有些"东西"既是对象又有属性,同时还能像函数一样被调用——典型例子是 JS 里 Date.now() 不行、但 Date() 会得到一个字符串的旧 API,以及很多工具库返回的"可调用对象"。

TS 用调用签名表达这种"既可调用又有属性"的形态:

// 写法: { 属性; (参数): 返回值 }
type DescribableFn = {
  description: string;
  (arg: number): boolean;
};

function doSomething(fn: DescribableFn) {
  console.log(fn.description);  // 访问属性
  console.log(fn(42));           // 像函数一样调用
}

和"函数类型表达式"对比着看:

// 函数类型表达式:只能调用,没有"属性"位置
type Fn1 = (x: number) => boolean;

// 调用签名:既能调用,也能挂属性
type Fn2 = {
  description: string;
  (x: number): boolean;
};

何时用:写框架(中间件、装饰器)、或给"既有属性又能调用"的第三方对象写类型时。普通业务函数优先用函数类型表达式,简单清楚。

来源:Handbook · Call Signatures。

6 · 函数重载 overloads:同一种行为,多种调用形态

有些函数对不同输入会给出不同输出——比如"传字符串就解析成日期,传时间戳数字就包装成 Date"。我们希望 TS 在调用点根据入参形状给出精确的返回类型。

这就是函数重载的舞台。它由多个重载签名 + 一个实现签名组成:

// 重载签名(对外可见,会出现在 .d.ts / 悬浮提示里)
function makeDate(timestamp: number): Date;
function makeDate(y: number, m: number, d: number): Date;

// 实现签名(对外不可见,只对实现代码可见;参数类型必须兼容所有重载)
function makeDate(yOrTs: number, m?: number, d?: number): Date {
  if (m !== undefined && d !== undefined) {
    return new Date(yOrTs, m - 1, d);
  }
  return new Date(yOrTs);
}

makeDate(1234567890);                  // ✅ 匹配第一个重载,返回 Date
makeDate(2026, 7, 15);                 // ✅ 匹配第二个重载,返回 Date
makeDate("2026-07-15");              // ❌ 任何重载都不匹配

常见坑 ③:重载的实现签名对外不可见

把鼠标悬停到 makeDate 上,你会看到 TS 只暴露两个重载签名,实现签名里的 m?: number, d?: number 在调用方眼里不存在。 所以:

  • 调用方写不出 makeDate(2026, 7) 这种"部分参数"调用(因为没有重载签名这么定义);
  • 实现签名里的参数必须兼容所有重载(通常用 union 或最宽类型),否则 tsc 报错;
  • 实现签名的返回类型必须能被所有重载签名的返回类型"赋值兼容",否则也报错。

经验:重载签名按"最具体 → 最宽"顺序写,TS 自上而下匹配。

何时用重载 vs union 参数

不是所有"多种输入"都需要重载。判断标准:

场景推荐原因
不同输入对应不同返回类型重载调用方在调用点就能拿到精确类型
不同输入但返回类型一样union 参数签名更短,无需维护多份
逻辑复杂、组合参数多重载 + 实现签名把"对外契约"与"实现"分开

来源:Handbook · Overload Signatures and the Implementation Signature。

7 · this 参数类型(简介)

JS 里 this 在普通函数中由"怎么调用"决定——常常踩坑。 TS 允许你在参数列表最前面显式声明 this 的类型,它不占位置、编译时也不出现在 JS 参数里:

interface User { name: string }
function intro(this: User) {
  return `I'm ${this.name}`;
}

const u: User = { name: "Ada", intro: intro };
u.intro();             // ✅ this 推断为 User
intro();               // ❌ this 是 undefined(strictBindCallApply 下会报错)

何时用:写"会被当方法调用"的工具函数、类库、回调时,把 this 类型锁死,可以避免大量运行时 Cannot read property 'x' of undefined。

来源:Handbook · Declaring this in a Function。

10 · 小结与下一步

  • ✅ 函数类型表达式:(a: T) => U,可赋给变量、参数、返回值。
