TypeScript 本质、类型注解与推断

本课小胜利：能准确说出 TS 到底做了什么、写出各种基本类型注解、并判断编译器为你推断出的类型——这是后面所有课的地基。约 18 分钟。

1 · TS 到底是什么：一个静态类型检查器

TypeScript 官方对自己的定义是:“a static typechecker for JavaScript programs — a tool that runs before your code runs.” 拆开看三个关键词:

词含义
static 静态在代码运行前分析,不需要真的执行
typechecker 类型检查检查“值的类型”和“用法”是否匹配
for JavaScript 面向 JS它是 JS 的超集:合法 JS 一定是合法 TS

关键心智模型:类型在编译后会被完全擦除(type erasure)。TS 编译产物就是普通 JS,浏览器/Node 运行的是那份 JS,类型只在“写代码时”帮你。

来源:TS for the New Programmer · The TypeScript Handbook。

为什么值得学:一个 JS 里“静默出错”的例子

// —— 纯 JavaScript,完全合法,不报错 ——
function total(items) {
  return items.reduce((sum, x) => sum + x.price, 0);
}
total({ price: 10 }); // 期望传数组,却传了对象 → 运行时才炸,或悄悄算错

同样的代码加上类型后,写代码的当下编辑器就会画红线:

function total(items: { price: number }[]): number {
  return items.reduce((sum, x) => sum + x.price, 0);
}
total({ price: 10 }); // ❌ 对象不能赋给数组

这就是核心价值:把“运行时才发现的 bug”提前到“敲代码时发现”。

2 · 类型注解的写法:变量、函数

注解的通用形状是 标识符: 类型。三种基本类型 string / number / boolean:

let name: string = "Ada";
let age: number = 30;      // 整数、小数、bigint 都算 number 家族
let active: boolean = true;

函数在参数和返回值两处标注:

function add(a: number, b: number): number {
  return a + b;
}
const greet = (who: string): string => `Hi ${who}`;  // 箭头函数同理

常见坑 ①:隐式 any

函数参数不写类型时,若开启了 strict,TS 会报 “Parameter implicitly has an 'any' type”。 因为 TS 无法从外部推断参数类型——参数必须显式标注。

function shout(msg) {          // ❌ msg 隐式为 any
  return msg.toUpperCase();
}

3 · 类型推断 Inference:少写却仍安全

当变量声明即初始化时,TS 自动推断类型,你不必手写:

let total = 42;        // 推断: number
const pi = 3.14;        // 推断: 3.14 (字面量类型!const 更窄,见 0002)
let names = ["a", "b"]; // 推断: string[]
total = "x";             // ❌ string 不能赋给 number

let 与 const 的推断不同:const 值不会变,所以被推断成更精确的字面量类型(下一课展开)。

经验法则: 变量能推断就别手写(减少噪音); 函数参数必须写; 函数返回值通常可省,但对外 API 建议显式写以锁定契约。 来源:Everyday Types。

6 · 跑起来:编译流程 & strict

最小心智模型(细节留到阶段四):

# 安装(项目内)
npm i -D typescript
# 一份最小配置
npx tsc --init
# 编译:app.ts → app.js
npx tsc

tsconfig.json 里最重要的一项是 "strict": true——它打开一组严格检查(含“禁止隐式 any”“空值检查”)。 本课程所有示例都假设 strict 开启,这也是业界默认。

不想装环境?用官方 Playground,浏览器里即时编译、看推断(把鼠标悬停到变量上会显示类型!)。
