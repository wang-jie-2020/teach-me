字面量类型、联合类型与类型别名

本课小胜利：能让一个变量、参数或返回值被锁死在“一组固定的值”之内， 并能用 type 给这组值命名复用——这是写“领域模型”和“穷举枚举”的第一块砖。约 20 分钟。

1 · 字面量类型 literal types：把“值”当“类型”用

之前 string、number、boolean 表示“任意字符串/数字/布尔”。 但很多时候你想要的更严格——只允许那几个具体值。 这时可以把字面量(literal, 即源码里直接写出来的值)本身当作类型:

let direction: "up" | "down" = "up";   // 字面量联合(稍后展开)
let lucky: 7 = 7;                              // 仅允许 7
let on: true = true;                       // 仅允许 true

三种字面量都能作类型:字符串字面量类型(string literal)、数字字面量类型(number literal)、布尔字面量类型(boolean literal)。 它们各自也是类型,可以参与联合。

来源:Handbook · Literal Types。

2 · 联合类型 union |：只能是这几个值之一

用竖线 | 把若干类型串起来,就得到一个联合类型(union type,或称“或类型”)—— 变量的值只需符合其中任意一个即可。最常见的写法是字符串字面量联合, 在业务里经常用来表达“枚举值”、“状态机”、“动作指令”等有限集合:

type Dir = "up" | "down" | "left" | "right";

function move(d: Dir, step: number) { /* ... */ }

move("up", 1);          // ✅
move("forward", 1);    // ❌ "forward" 不在 Dir 里

联合不限于字面量,任意类型都能联合:

type Result = string | number;          // 字符串或数字
type Maybe = { id: number } | null;     // 对象或 null(配合收窄)

来源:Handbook · Union Types。

3 · 类型别名 type alias：给一个类型起名字

字面量联合写一遍还行,到处复制就难受。type 关键字可以把任意类型表达式(基本类型、联合、对象、函数……)起一个名字复用:

type Dir = "up" | "down" | "left" | "right";
type UserId = string;                     // 别名,不是新类型
type Point = { x: number; y: number };      // 对象类型也能别名
type Handler = (e: MouseEvent) => void;     // 函数类型别名

注意一个常见误解:别名只是“名字”,不是新类型。 type UserId = string 不会让 UserId 跟 string 不兼容—— 它们在结构上是同一个类型(详见面 vs 标的区别要到阶段二)。

对象类型别名更多写法见 Handbook · Object Types。

4 · 字面量联合 = 轻量“穷举枚举”模式

很多语言需要先声明 enum 才能表达“只能取这些值”。TS 里, 字符串字面量联合 + type 别名 是同等常见、且更轻量的替代:

type Status = "idle" | "loading" | "success" | "error";
type Color  = "red"  | "green" | "blue";
type Method = "GET"  | "POST" | "PUT"  | "DELETE";

这种写法的好处是“结构透明、零运行时成本”——它就是字符串,没有额外对象。 后续会用它配合 switch 做穷尽性检查(exhaustiveness)。

TS 也提供真正的 enum 关键字, 留到阶段二 0005 详细对比(语义、值、编译产物都不一样)—— 这里先把字面量联合的“穷举”用法练熟。

5 · 常见坑 ①:字面量 widening(拓宽)

字面量为何“消失”?let vs const 的推断差异

这是这一节最常踩的坑。回到 0001 第 3 节的一个例子:

let  a = "north";   // 推断: string  ← 被拓宽了!
const b = "north";   // 推断: "north"  ← 仍是字面量类型

原因:TS 默认会把 let 声明的字面量拓宽(widening)到它的基础类型,因为这个变量将来可能赋别的值; const 声明的不能被重新赋值,所以被保留为更精确的字面量类型。

这在“希望它就是这几个字面量之一”时是个问题:

type Dir = "up" | "down" | "left" | "right";

let heading = "up";        // 推断: string
move(heading, 1);               // ❌ string 不能赋给 Dir

修法有三种,推荐前两种:

  1. 用 const 声明:const heading = "up"; → 字面量类型保留。
  2. 显式注解:let heading: Dir = "up";
  3. as const 断言:把整个表达式“冻结”为字面量(对象/数组里尤其常用):
const dirs = ["up", "down"] as const;     // readonly ["up", "down"]
const cfg  = { mode: "strict" } as const; // { readonly mode: "strict" }

6 · 联合类型的“使用门槛”:必须先收窄

当一个变量是联合类型时,只有所有分支共有的成员可以直接用:

type Pet = { kind: "dog"; bark: () => void }
      | { kind: "cat"; meow: () => void };

function talk(p: Pet) {
  p.bark();   // ❌ 只有 cat 分支有 meow,dog 没有 bark
}

必须先把“模糊的 union”收窄(narrow)到具体一个分支才能用其独有成员。 典型做法是看判别字段(上面是 kind):

function talk(p: Pet) {
  if (p.kind === "dog") p.bark();   // ✅ 这里 p 被收窄到 dog 分支
  else                  p.meow();   // ✅ 收窄到 cat 分支
}

收窄(narrowing)的内容非常丰富——typeof、in、instanceof、判别字段、 控制流分析……整节课讲,下一课 0003 见。

承上启下:下一课 0003 · 类型收窄。
