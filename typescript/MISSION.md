# Mission: 掌握 TypeScript，作为职业技能

## Why
把 TypeScript 变成可以写进简历、能在工作中独立使用的技能。
用户已有一些 JavaScript 基础，目标是从“会一点 JS”成长为“能自信地用 TS 写出类型安全、可维护的代码”，从而在求职和日常开发中更有竞争力。

## Success looks like
- 能读懂并写出常见类型注解（primitives、数组、对象、函数、union、可选属性）
- 能用 `interface` / `type` 描述数据结构，并让编译器帮自己抓错
- 能理解并利用类型推断（inference），知道何时该显式标注、何时可以省略
- 能配置一个最小可运行的 TS 项目（tsc / tsconfig）并修复类型报错
- 能在面试中解释 TS 相比 JS 的价值，并现场写出基础类型代码

## Constraints
- 已有“一些 JavaScript”基础，但还不完全自信 —— 讲解要补齐必要的 JS 缺口，不假设精通
- 偏好：动手写代码 + 清晰讲解 + 可回顾的速查表 + 真实项目
- 尚未确定前端/后端方向 —— 优先教“到处都用得上”的 TS 核心，暂不深绑定框架
- 每节课要短、可快速完成，并给出即时反馈

## Out of scope（当前阶段）
- 高阶类型体操（conditional types、mapped types、复杂泛型推断）暂缓
- 特定框架深度绑定（React/Angular/Node 专属类型）暂缓，等方向明确
- 编译器内部原理、装饰器元编程等暂不涉及
